2. CAMERA SHOULD ZOOM OUT A BIT
- Zoom the gameplay camera out so the player can see more of the play space
- Make planets appear a bit smaller on screen by adjusting camera zoom and/or planet scale

3. PREVIOUS PLANET SHOULD NOT CATCH THE PLAYER AGAIN
- Once the player launches from a planet to the next one, they should not be able to reattach to the previous planet
- They should pass through the previous planet instead
- Implement logic so only the current valid target planet can be landed on
- The previously used planet should either:
  - disable its landing collider / landing logic for the player
  - move to a non-interactable state
  - or no longer count as a valid landing body
- This is very important because the player should always be progressing forward

4. SMALL SLING PULL SHOULD RESULT IN SHORTER DISTANCE, NOT JUST SLOWER TRAVEL
- Right now when the player pulls back only a little, the rocket still goes too far and just moves more slowly
- Fix the launch physics so launch distance is directly tied to pull strength
- A small pull should create a genuinely short hop
- A medium pull should create a medium jump
- A large pull should create a long jump
- The rocket should not glide forever from a weak input
- Adjust force, damping, velocity behavior, gravity influence, and/or travel model so this feels correct
- The controls should feel skill-based and intuitive
